# Student Information System (SIS)

A robust, desktop-based management application built with Python, PyQt6, and MySQL. This system provides a comprehensive interface for managing student records, academic programs, and college departments with full CRUD (Create, Read, Update, Delete) capabilities and data integrity.

## Features

* Student Management: Track student IDs, names, gender, year levels, and program affiliations.
* Academic Hierarchy: Manage Colleges and Programs with automatic cascading logic (handling NULL references if a parent entity is deleted).
* Dynamic Search & Filtering: Real-time search across all tables and dropdown-based sorting.
* Custom Pagination: Efficiently handles large datasets (tested up to 5,000+ records) with "Jump to Start/End" functionality.
* Relational Data Integrity: Built-in validation to ensure Student IDs follow specific formats (YYYY-NNNN) and prevent duplicate entries.
* Modern UI: Clean, responsive interface featuring custom CSS styling and a `QStackedWidget` for seamless navigation.

## Tech Stack

* Frontend: PyQt6 (Python Bindings for Qt6)
* Backend: Python 3.x
* Database: MySQL
* UI Design: Qt Designer (`.ui` files)

## Prerequisites

Before running the application, ensure you have the following installed:
* [Python 3.8+](https://www.python.org/)
* MySQL Connector for Python

## 🔧 Installation & Setup

1.  **Clone the Repository**:
    ```bash
    git clone https://github.com/yourusername/student-management-system.git
    cd student-management-system
    ```

2.  **Install Dependencies**:
    ```bash
    pip install PyQt6 mysql-connector-python
    ```

3.  **Database Configuration**:
    * Start **Apache** and **MySQL** in your XAMPP Control Panel.
    * Open **phpMyAdmin** and create a database named `student_db` (or as specified in your `config.py`).
    * Import the provided SQL schema or run the table creation queries for `colleges`, `programs`, and `students`.

4.  **Run the Application**:
    ```bash
    python main.py
    ```

## Project Structure

* `main.py`: Entry point and UI logic for the Main Window.
* `database_manager.py`: Handles all SQL queries and MySQL connections.
* `models.py`: Contains data validators and business logic.
* `main_window.ui`: The XML layout file generated by Qt Designer.
* `config.py`: Database credentials and configuration.

## Database Schema

The project follows a relational structure:
* **Colleges**: `code` (PK), `name`
* **Programs**: `code` (PK), `name`, `college_code` (FK)
* **Students**: `id` (PK), `firstname`, `lastname`, `year`, `gender`, `program_code` (FK)
